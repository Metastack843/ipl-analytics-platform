### 1. Data Cleaning (`01_data_cleaning.ipynb`)

- **Standardization**: Unifies team names (e.g., _Delhi Daredevils_ â†’ _Delhi Capitals_).
- **Derivation**: Creates binary `is_wicket` column from dismissal types.

### 2. Feature Engineering (`02_feature_engineering.ipynb`)

#### **Key Metrics Implemented**

- **Consistency Index** (Player Level):
  $$ \text{Consistency Index} = \frac{\text{Avg Runs per Match}}{\text{Std Dev of Runs}} $$
  _Higher is better. Identifies reliable anchors vs. volatile hitters._

- **Match Intensity Score** (Team/Match Level - Fan Engagement Proxy):
  $$ \text{Intensity} = \text{Total Runs} + (\text{Wickets} \times 20) + (\text{Boundaries} \times 5) $$
  _Quantifies the excitement level of a match._

- **Team Performance**:
  - Win Rate (%)
  - Home vs. Away Win % (Enriched with venue data)
